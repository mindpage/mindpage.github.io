会员支付组件

该支付组件主要用于活动中售卖微博会员、联合会员。不适合其他支付行为，如其他业务支付、V+支付、收银台外露形式。

配置项说明

售卖项

  • 连续和非连续：售卖项分为连续和非连续，主要用于用户切换自动续费时，为用户提供不同的收买想，如果相同，则设置相同的信息。
  • 售卖项数量：支持单个、2个、3个，通过点击售卖项和售卖项(连续)模块下的「添加字块」即可增加，也可删除、更改顺序。目前支持设置3个以上的收买想，但是样式有问题。后续增加对于三个的样式展示。
  • 单个售卖项：包含id、售卖图（选中、非选中）
    • subitemid:微博会员产品对应的id
    • productid:联合会员收银台对应的id
    • 售卖图:如果有多个售卖项的时候，需要提供选中和非选中两种状态的图片，如果不提供用户选择时没有明确UI提示。如果只有一个售卖项，则填相同的即可。

示例

  • 只有单个售卖项，连续和非连续信息一样、选中和非选中图片一样。
  • 多个售卖项。

F值

支付策略对应设置的F值，也对应支付后要下发的权益。需要在admin后台配置，可联系linzhong@staff.weibo.com

支付前接口

支付前用户是否能参与支付的校验接口，需要单独开发，将接口地址填写在这里即可，需要绝对路径，如：https://new.vip.weibo.cn/beforepay。

常用支付前接口：后续会提供常用的支付前校验接口，可根据需求自行选择。也欢迎业务方共享接口。

  • 敬请期待
    • 接口地址：敬请期待
    • 使用场景：敬请期待

是否连续购买

设置是否默认选中连续购买选项，如果不填写则不选中。

字体颜色

设置自动续费文本和支付方式选择框外边框的颜色，例如：#ff0717

按钮

可以使用图片，也可以使用标准样式，更改按钮文案

  • 使用图片：上传设计好的按钮图片即可，目前暂不支持png格式，紧急开发中。如果使用图片按钮，标准样式设置则失效。
  • 标准样式：圆角按钮，可更改「按钮文字」。
